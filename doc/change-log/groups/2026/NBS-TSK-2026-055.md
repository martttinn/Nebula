# Resumen

Grupo que recoge la adopción de la primitive de botones de `animate-ui` como base técnica compartida para todos los botones reutilizables del proyecto.

## Objetivo del grupo

Unificar interacción, `hoverScale` y `tapScale` entre CTAs estándar y premium sin perder el styling Nebula ya definido.

## Cambios incluidos

- import de la primitive `animate-ui` desde `shadcn`
- refactor de `components/ui/button.tsx` para apoyarse en la nueva base
- integración de la misma primitive dentro de `StarBorder`
- actualización del canon documental y de la referencia técnica

## Impacto en arquitectura / producto

Los botones del hero y el CTA del navbar comparten ya una base motion única, lo que reduce drift de interacción entre primitives distintas.

## Validación agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- al convivir `framer-motion` y `motion`, conviene vigilar peso y solapamientos si en el futuro se amplía mucho el uso de `animate-ui`

## Notas para presupuesto

Refactor controlado de primitive reusable con impacto transversal en UI pública.
