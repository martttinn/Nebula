# Resumen

Grupo que recoge la retirada de la fachada de compatibilidad de `services-carousel` para que la home resuelva directamente la carpeta canónica de la sección.

## Objetivo del grupo

Seguir cerrando la transición desde las fachadas temporales hacia el consumo directo de la arquitectura por carpetas ya implantada.

## Cambios incluidos

- retirada de `components/home/services-carousel.tsx`

## Impacto en arquitectura / producto

La home mantiene el mismo specifier de import, pero ahora este resuelve directamente `components/home/services-carousel/index.tsx`.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si el editor tenía abierta la fachada antigua, habrá que reabrir la sección desde la carpeta nueva
- la misma limpieza de compatibilidad sigue pendiente en `how-we-work` y en la familia `hero`
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Paso corto de consolidación arquitectónica posterior a la modularización del carrusel de servicios.
