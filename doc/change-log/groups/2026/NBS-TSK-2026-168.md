# Resumen

Grupo que recoge la correccion de los cuellos de botella detectados en Vercel Speed Insights para la home publica de Nebula Studios.

## Objetivo del grupo

Reducir los riesgos principales de `LCP` y `CLS` de campo sin redisenar la identidad visual ni introducir backend o dependencias nuevas.

## Cambios incluidos

- ajuste del `Preloader` para acortar la retencion visual inicial y adelantar la secuencia del hero
- render inmediato del heading principal del hero para evitar que el elemento LCP dependa de un reveal oculto
- retirada de `HeroParticles` en stages sticky/fixed de servicios, proceso y testimonials
- carga bajo demanda de `GridDistortion` y de su imagen pesada en la card final de contacto
- actualizacion de `DESIGN.md`, decisions log y referencia tecnica

## Impacto en arquitectura / producto

Impacto frontend visual y runtime en la landing publica. No cambia copy comercial, rutas, SEO estructural, backend, auth, datos ni integraciones externas. El ajuste desplaza ornamento caro fuera del primer render y reduce capas decorativas con impacto medible en estabilidad visual.

## Validacion agregada

- documentacion oficial de Vercel Speed Insights consultada el `2026-05-18`
- `curl -I https://somosnebula.com`
- Lighthouse local desktop antes y despues
- Playwright CLI con `PerformanceObserver` de `layout-shift` durante scroll sintetico
- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npx react-doctor@latest --offline --full --no-lint --json --json-compact`
- `npm run changes:sync`

## Riesgos o arrastres

Las metricas reales de Vercel no cambian hasta desplegar y acumular nuevas visitas reales. La muestra actual del panel era pequena para desktop, por lo que los percentiles pueden moverse de forma brusca. Queda como optimizacion separada estudiar el chunk lazy de `tech-stack-icons`, que es muy grande aunque no aparezca como causa directa del primer render medido.

## Notas para presupuesto

Trabajo facturable de performance operativo: diagnostico con datos de campo, medicion local reproducible, cambios acotados de runtime y documentacion de canon para evitar regresiones. El valor es visible para usuario final por menor espera inicial y menor inestabilidad durante scroll, aunque parte del beneficio se confirma solo con datos de campo tras deploy.
