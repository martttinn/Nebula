# Resumen

Grupo que recoge el ajuste de padding lateral dentro de cada wrapper del timeline desktop de `How we work`.

## Objetivo del grupo

Evitar que nodos y cards queden demasiado pegados a los laterales del viewport, manteniendo al mismo tiempo el cruce de la linea SVG por los hitos reales.

## Cambios incluidos

- anadido padding horizontal a cada `DesktopRow` de `components/home/how-we-work.tsx`

## Impacto en arquitectura / producto

No cambia contratos ni estructura de la home. Es una calibracion local del layout desktop de una seccion publica existente.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- el ajuste puede requerir una segunda calibracion si cambian el ancho objetivo de las cards o el recorrido deseado del path
- `npm run changes:sync` sigue sin poder ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Refinamiento puntual de layout y respiracion lateral en una seccion publica.
