# Resumen

Grupo que recoge la incorporación de un dropdown de servicios sobre el link `Servicios` del navbar desktop, usando cuatro cards cuadradas y redondeadas alimentadas desde el catálogo público de servicios.

## Objetivo del grupo

Hacer que la navegación principal anticipe el alcance de Nebula Studios sin obligar al usuario a bajar hasta la sección de servicios para entender las capacidades disponibles.

## Cambios incluidos

- incorporación del dropdown hover en `components/layout/navbar.tsx`
- reutilización del catálogo `data/services.ts`
- actualización de `DESIGN.md`, referencia técnica, `decisions-log` y `change-log`

## Impacto en arquitectura / producto

El navbar desktop deja de ser una lista plana de links y gana una affordance informativa premium sobre `Servicios`. La navegación reutiliza el mismo catálogo público que la sección de servicios y evita abrir una taxonomía paralela.

## Validación agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- no se ha hecho todavía revisión visual automatizada del dropdown en navegador para calibrar distancia, blur y escape de hover
- `npm run changes:sync` sigue sin poder ejecutarse porque ese tooling no está expuesto en `package.json`

## Notas para presupuesto

Entrega visible de frontend con mejora de navegación y reutilización del mismo catálogo de servicios de la home.
