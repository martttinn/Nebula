# Resumen

Grupo que recoge la recuperación del patrón `hide on scroll` del navbar desde `nebula-legacy`.

## Objetivo del grupo

Reducir la presencia visual del navbar durante la lectura y recuperar una reaparición inmediata al remontar, manteniendo consistencia entre desktop y responsive.

## Cambios incluidos

- conversión de `components/layout/navbar.tsx` a componente cliente para observar `window.scrollY`
- réplica del patrón histórico de Nebula: visible cerca del top, oculto al descender y visible otra vez al remontar
- elevación del estado del menú responsive para impedir que la cabecera desaparezca mientras el panel está abierto
- saneamiento del `change-log` para separar este cambio de navegación del grupo de seams móviles con el que había quedado mezclado por colisión de ID

## Impacto en arquitectura / producto

La navegación global se comporta de forma más contenida y más alineada con el patrón histórico del proyecto, y el histórico documental vuelve a reflejar correctamente la naturaleza del cambio.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- la entrega original no incluyó revisión visual manual en navegador, por lo que el ajuste fino de timing y distancia de ocultación dependía de runtime real

## Notas para presupuesto

Entrega visible de navegación y motion con impacto transversal en la shell pública.
