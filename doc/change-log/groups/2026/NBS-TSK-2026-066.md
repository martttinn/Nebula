# Resumen

Grupo que recoge la correccion del comportamiento sticky de la seccion de servicios en runtime.

## Objetivo del grupo

Hacer que el sticky interno del carrusel de servicios pueda fijarse correctamente al viewport en desktop.

## Cambios incluidos

- retirada de `overflow-hidden` del `section` exterior de servicios para no bloquear `position: sticky`

## Impacto en arquitectura / producto

La mecanica del carrusel no cambia, pero el sticky interno deja de quedar condicionado por un ancestro con overflow no visible.

## Validacion agregada

- `npm run lint`
- `npm run build`

## Riesgos o arrastres

- no se ha hecho revision visual automatizada en navegador en esta sesion

## Notas para presupuesto

Correccion puntual de layout y comportamiento CSS sobre una surface publica ya integrada.
