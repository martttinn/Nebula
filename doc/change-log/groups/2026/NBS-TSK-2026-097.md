# Resumen

Grupo que recoge la activacion temporal de bordes de debug en la seccion `How we work` de la home publica.

## Objetivo del grupo

Hacer visible la caja real de cada capa del layout para revisar alineaciones de cards, canal central y estructura mobile sin alterar la geometria de la seccion.

## Cambios incluidos

- activacion de un flag local `DEBUG_BORDERS` en `components/home/how-we-work.tsx`
- aplicacion de `outline` de colores por capas sobre section, shell, cabecera, grid, celdas, canal SVG y cards desktop/mobile
- aislamiento del debug dentro de la propia seccion para no contaminar otras surfaces
- apertura del wrapper desktop del timeline a `100%` de ancho util del shell y retirada de wrappers lila redundantes del propio timeline

## Impacto en arquitectura / producto

No cambia contratos ni estructura del repo. Solo introduce una ayuda visual temporal en runtime para inspeccion de layout en una seccion concreta.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- los bordes debug son visibles en runtime mientras el flag siga activo y no deben confundirse con styling definitivo
- `npm run changes:sync` sigue sin poder ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Ajuste puntual de frontend para inspeccion visual y depuracion de layout en una seccion publica concreta.
