# Resumen

Grupo que recoge el replanteamiento visual y funcional de la seccion `How we work` para alinearla con una referencia de timeline serpenteante que cruza la pantalla por scroll.

## Objetivo del grupo

Sustituir el timeline anterior por una composicion mas cinematica y editorial: una linea SVG que entra desde el lateral, conecta nodos con icono y despliega cada card un poco antes de alcanzar su punto correspondiente.

## Cambios incluidos

- reescritura completa de `components/home/how-we-work.tsx`
- nuevo path SVG full-width medido sobre nodos reales con `ResizeObserver`
- animacion de recorrido por scroll con la linea entrando desde el lateral y saliendo por el extremo opuesto
- cards de proceso reveladas de forma anticipada respecto al avance de la linea
- mantenimiento de una variante movil simplificada y legible
- actualizacion de `doc/reference/technical-reference.md`

## Impacto en arquitectura / producto

No cambia contratos de datos ni estructura global de la home, pero si cambia de forma relevante la primitive visual y de motion de una seccion publica de marca.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- el ajuste fino del recorrido del path puede requerir nueva calibracion visual cuando se contraste contra nuevas referencias
- `npm run changes:sync` sigue sin poder ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Rediseño de una seccion publica con trabajo combinado de layout, motion, medicion runtime y ajuste documental.
