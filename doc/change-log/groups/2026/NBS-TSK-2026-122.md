# Resumen

Grupo que recoge el refuerzo del reveal visual de los nodos ilustrados en la sección de proceso.

## Objetivo del grupo

Hacer que la activación de cada cluster se sienta más intencional mediante más escala y una recuperación progresiva de opacidad en los nodos.

## Cambios incluidos

- ampliación del rango de escala del nodo
- reducción de opacidad base en reposo para imágenes
- sincronización del timing con la aparición de la card
- actualización del canon visual en `DESIGN.md`

## Impacto en arquitectura / producto

No cambia la estructura del timeline ni su lógica geométrica. Ajusta el comportamiento visual del reveal de una primitive pública clave.

## Validacion agregada

- `npm run lint`
- `npm run build`

## Riesgos o arrastres

- si el gesto acaba viéndose demasiado enfático en algunos breakpoints, convendrá recalibrar `NODE_REVEAL_BASE_SCALE` o `IMAGE_NODE_BASE_OPACITY`
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Pulido visual incremental con impacto directo en la percepción de calidad del recorrido de proceso.
