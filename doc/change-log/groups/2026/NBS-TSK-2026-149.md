# Resumen

Grupo que corrige la repetición de posiciones en los `abstract-icons` de la banda de benefits y formaliza la unicidad de anclajes como regla del catálogo ornamental.

## Objetivo del grupo

Eliminar anclas ornamentales repetidas entre frases distintas de la propuesta de valor para que cada statement conserve una composición espacial propia.

## Cambios incluidos

- reajuste de posiciones desktop y mobile en `components/home/value-proposition-section/content.ts`
- validación explícita contra duplicados de anclaje por viewport dentro del catálogo ornamental
- actualización del canon visual y de la referencia técnica para reflejar la nueva restricción

## Impacto en arquitectura / producto

Impacto local pero visible en una superficie pública de alto peso editorial: mejora la percepción de cuidado compositivo y endurece el contrato de datos visuales sin añadir complejidad de runtime relevante.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm run changes:sync`

## Riesgos o arrastres

- la validación protege contra duplicados exactos, no contra parecidos compositivos más subjetivos
- el efecto final sigue dependiendo de revisión visual manual en viewport real, especialmente en móvil

## Notas para presupuesto

Entrega visible de ajuste fino con componente de hardening: refuerza la calidad percibida de la home y evita regresiones futuras en una primitive ornamental sensible.
