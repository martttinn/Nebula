# Resumen

Grupo que recoge la extracción del hero de la home a un componente dedicado y mínimo.

## Objetivo del grupo

Crear `components/home/hero.tsx` y dejar el hero, por ahora, reducido a un bloque fullscreen con `GridScan`.

## Cambios incluidos

- creación de `components/home/hero.tsx`
- simplificación de `app/page.tsx` para delegar la home en `Hero`
- validación estática y de build

## Impacto en arquitectura / producto

La home gana una boundary más limpia para futuras iteraciones visuales, sin añadir todavía contenido ni complejidad adicional dentro del hero.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- el hero queda intencionalmente vacío salvo por el fondo, así que la home no comunica contenido todavía
- cualquier overlay futuro deberá reconstruirse sobre esta nueva boundary

## Notas para presupuesto

Trabajo de estructura frontend orientado a preparar la home para iteraciones posteriores sin acoplar el layout directamente a `app/page.tsx`.
