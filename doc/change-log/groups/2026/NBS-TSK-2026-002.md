# Resumen

Grupo que recoge la auditoría y normalización del bootstrap de Supabase en Nebula Studios.

## Objetivo del grupo

Dejar la instalación de Supabase alineada con el patrón actual de SSR para Next.js, añadir el directorio `supabase/` con soporte de migraciones y corregir el drift mínimo necesario para validar el repo.

## Cambios incluidos

- revisión comparativa contra documentación oficial de Supabase y otros proyectos Next.js locales
- normalización de `lib/supabase/**` con helper de entorno y flujo SSR actual
- introducción de `supabase/config.toml`, `supabase/seed.sql` y `supabase/migrations/`
- adición del CLI de Supabase como dependencia de desarrollo y scripts operativos mínimos
- corrección del helper de `siteUrl` necesario para que `robots.ts` y `sitemap.ts` compilen

## Impacto en arquitectura / producto

Se incorpora una base repo-safe para evolución futura de auth y esquema sin presentar todavía un backend live como hecho cerrado.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- el repo sigue en Next.js 14; la instalación de Supabase queda correcta para esta base, pero el upgrade del framework sigue pendiente si se quiere converger con otros proyectos en Next 16
- siguen faltando scripts documentados del `change-log` en `package.json`
- no existe todavía un proyecto Supabase live vinculado ni un esquema real versionado

## Notas para presupuesto

Trabajo estructural de infra y hardening: bajo impacto visual inmediato, alto retorno para evitar drift cuando se active backend real.
