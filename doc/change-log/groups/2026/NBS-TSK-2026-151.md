# Resumen

Grupo que recoge la sustitución del fondo del hero por `DotField` y la depuración posterior de su atmósfera visible.

## Objetivo del grupo

Reemplazar el antiguo fondo del hero por una implementación canvas basada en React Bits y dejar la cabecera más limpia sin romper la orquestación actual del preloader.

## Cambios incluidos

- integración repo-local de `DotField` dentro de `components/home/hero/`
- retirada del runtime `GridScan` y borrado de sus archivos canónicos del hero
- preservación del evento `hero-grid-ready` para coordinar el `Preloader`
- eliminación del radial decorativo del overlay del hero
- retirada de `HeroParticles` del hero manteniendo la primitive disponible para otras secciones
- actualización de contexto operativo, `decision log`, referencia técnica y `change-log`

## Impacto en arquitectura / producto

El above-the-fold pasa de un fondo WebGL a una primitive canvas más contenida y el hero reduce ruido atmosférico sin perder la secuencia de entrada existente. El cambio afecta a la percepción visual inmediata de la home y deja el runtime documentado de forma coherente con el código real.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm run changes:sync`

## Riesgos o arrastres

- la limpieza se ha acotado al runtime del hero: no se han retirado dependencias visuales antiguas del repo porque el worktree ya venía sucio y esa poda no formaba parte del scope pedido
- queda pendiente, si lo quieres, una revisión visual manual en navegador para ajustar densidad, contraste o interacción del `DotField` después de verlo en desktop y móvil

## Notas para presupuesto

Trabajo de refinamiento frontend visible sobre la cabecera principal, con sustitución de primitive, preservación de contratos de carga y sincronización documental repo-local.
