# Resumen

Grupo que recoge la reestructuración del footer para seguir el layout de referencia indicado por Martín, sin copiar su estética visual.

## Objetivo del grupo

Adaptar únicamente la estructura del footer a una fila superior de columnas y una barra inferior separada, manteniendo el lenguaje visual Nebula y colocando `Contactar` dentro de la sección de navegación.

## Cambios incluidos

- `components/layout/footer.tsx` pasa de tres columnas a una estructura desktop de cuatro columnas superiores.
- La primera columna concentra marca, claim y email de contacto confirmado.
- Las columnas secundarias quedan separadas como `Navegación`, `Estudio` y `Legal`, con `Contactar` dentro de `Navegación`.
- Se añade un separador horizontal y una barra inferior con copyright a la izquierda y legales a la derecha.
- `DESIGN.md` y `doc/reference/technical-reference.md` documentan la nueva estructura.

## Impacto en arquitectura / producto

El cambio es estructural y visual dentro del footer. No altera rutas, navegación principal, backend, formularios ni contenido comercial de alto impacto. Mantiene el catálogo compartido de navegación como fuente de verdad.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- smoke visual desktop y móvil en `http://localhost:3001`
- `npm run changes:sync`

## Riesgos o arrastres

Los links legales siguen desactivados porque sus páginas no existen. La barra inferior duplica esos destinos como en la estructura de referencia, manteniendo el estado desactivado para no fingir navegación operativa.

## Notas para presupuesto

Trabajo visible de UI y responsive acotado al footer, con actualización documental asociada y sin cambio de arquitectura de datos.
