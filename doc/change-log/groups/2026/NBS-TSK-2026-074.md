# Resumen

Grupo que recoge la integración de `Lenis` como capa global de smooth scroll en la web pública de Nebula.

## Objetivo del grupo

Suavizar el scroll del sitio sin romper anclas internas, accesibilidad ni scrolls anidados ya existentes.

## Cambios incluidos

- instalación de `lenis` como dependencia runtime
- integración global cliente en el `layout` raíz mediante `components/layout/lenis-provider.tsx`
- actualización del `Preloader` para bloquear también el scroll del `html`
- opt-out explícito con `data-lenis-prevent` en el panel del `StaggeredMenu` y en el carrusel móvil de servicios
- actualización del canon técnico y visual para reflejar el nuevo contrato de scroll

## Impacto en arquitectura / producto

La web pública gana una capa transversal de motion a nivel de runtime, pero la mantiene contenida: el scroll raíz se suaviza y las superficies con comportamiento propio conservan su semántica nativa.

## Validación agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- no se ha hecho revisión visual automatizada en navegador en esta sesión
- varios hashes del navbar siguen apuntando a secciones que hoy no existen en runtime, por lo que `Lenis` no puede animar esos destinos todavía

## Notas para presupuesto

Integración transversal de motion sobre el runtime público con impacto visible en la experiencia global y hardening específico para accesibilidad y scrolls anidados.
