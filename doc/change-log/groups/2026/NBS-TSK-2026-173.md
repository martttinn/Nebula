# Resumen

Grupo que recoge la incorporación del teléfono público confirmado de Nebula Studios como dato de contacto versionado y visible en el footer.

## Objetivo del grupo

Centralizar el teléfono `+34 622 028 550` junto al email operativo y exponerlo sin crear una integración comercial que todavía no existe.

## Cambios incluidos

- `lib/site.ts` añade `contactPhone` y `contactPhoneHref`.
- `components/layout/footer.tsx` muestra email y teléfono en la columna de marca/contacto.
- La documentación operativa y de dominio recoge el teléfono como canal comercial directo confirmado.
- El change-log queda sincronizado con la nueva entrega.

## Impacto en arquitectura / producto

El cambio mantiene el runtime público como landing estática. No añade rutas, APIs, formulario, calendario, CRM ni backend. El teléfono se modela como dato compartido para evitar hardcodes futuros.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- smoke visual desktop en `http://localhost:3001`
- smoke visual móvil en `http://localhost:3001`
- `npm run changes:sync`

## Riesgos o arrastres

El teléfono abre el marcador del dispositivo mediante `tel:`, por lo que no aporta tracking, atribución ni persistencia de leads. Si se quiere medir conversión telefónica hará falta una iteración específica.

## Notas para presupuesto

Entrega pequeña de contenido operativo y UI pública, con centralización del dato y actualización documental proporcional.
