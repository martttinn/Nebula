# Resumen

Grupo que recoge el ajuste del workspace de VS Code para evitar warnings falsos del validador CSS sobre directivas válidas de Tailwind.

## Objetivo del grupo

Reducir ruido operativo en el editor sin tocar el runtime ni alterar el pipeline de build del proyecto.

## Cambios incluidos

- adición de `.vscode/settings.json`
- desactivación local de `unknownAtRules` para CSS, SCSS y LESS en el workspace

## Impacto en arquitectura / producto

No hay impacto en runtime ni en arquitectura de frontend. Es una mejora de tooling local para que el editor no marque como errores reglas válidas como `@tailwind` y `@apply`.

## Validacion agregada

- revisión manual del contenido de `.vscode/settings.json`
- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- el ajuste vive solo en el workspace de VS Code; no cambia el comportamiento de otros editores ni del CLI

## Notas para presupuesto

Pequeño hardening operativo orientado a reducir fricción diaria de desarrollo y evitar falso ruido visual en revisiones de CSS.
