# Resumen

Grupo que recoge la integración de Vercel Web Analytics en la web pública de Nebula Studios.

## Objetivo del grupo

Dejar el runtime preparado para que el panel de Vercel empiece a contar visitantes y page views tras el siguiente despliegue.

## Cambios incluidos

- instalación de `@vercel/analytics`
- montaje de `<Analytics />` en `app/layout.tsx` junto a `SpeedInsights`
- actualización de referencia técnica y decisions log para reflejar la nueva capa de observabilidad pública

## Impacto en arquitectura / producto

Impacto runtime global y no visual. No añade rutas propias, backend, auth ni captación comercial; introduce instrumentación oficial de Vercel en la shell raíz de Next.js.

## Validacion agregada

- documentación oficial de Vercel Web Analytics consultada el `2026-05-18`
- `npm view @vercel/analytics version description --json`
- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm audit --omit=dev` falló por avisos existentes de `next`/`postcss`; no se aplicó fix porque requiere upgrade de framework fuera del scope
- `npm run changes:sync`

## Riesgos o arrastres

La recogida real de datos depende de deploy en Vercel y de que Web Analytics esté habilitado en el dashboard. Al trabajar sobre un worktree ya sucio, este grupo solo cubre los cambios específicos de Analytics y su documentación directa. Queda pendiente decidir un upgrade de Next/PostCSS para cerrar el audit productivo.

## Notas para presupuesto

Integración operativa de observabilidad con valor directo para medir tráfico real y validar adquisición, sin implementar un sistema propio de analítica ni ampliar backend.
