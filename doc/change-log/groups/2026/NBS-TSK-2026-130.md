# Resumen

Grupo que recoge la retirada de la fachada de compatibilidad de `how-we-work` para que la home resuelva directamente la carpeta canónica de la sección.

## Objetivo del grupo

Seguir cerrando la transición desde los wrappers temporales hacia el consumo directo de la nueva arquitectura modular.

## Cambios incluidos

- retirada de `components/home/how-we-work.tsx`

## Impacto en arquitectura / producto

La home mantiene el mismo specifier de import, pero ahora este resuelve directamente `components/home/how-we-work/index.tsx`.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si el editor tenía abierta la fachada antigua, habrá que reabrir la sección desde la carpeta nueva
- la misma limpieza de compatibilidad sigue pendiente en la familia `hero`
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Paso corto de consolidación arquitectónica para dejar la sección de proceso consumiendo ya su directorio canónico.
