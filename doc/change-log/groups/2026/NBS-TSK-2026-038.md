# Resumen

Grupo que recoge el ajuste de timing del hero para que el copy principal y los CTAs entren con más rapidez.

## Objetivo del grupo

Reducir la latencia percibida entre la salida del preloader y la aparición completa del bloque principal del hero.

## Cambios incluidos

- adelanto del trigger `hero-intro-start`
- reducción de delay y duración en heading y subheading
- compactación del `fade-up` de los CTAs

## Impacto en arquitectura / producto

La entrada del hero se siente más inmediata sin sacrificar la estructura secuencial del bloque principal.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- si se acelera demasiado en revisiones futuras, el riesgo principal será perder aire entre titular, supporting copy y CTA

## Notas para presupuesto

Refinamiento menor de timing e interacción inicial.
