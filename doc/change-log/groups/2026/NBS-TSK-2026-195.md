# Resumen

Grupo que recoge el rollback acotado del cambio que rompio el `BorderBeam` del CTA `Contactar` en el navbar desktop.

## Objetivo del grupo

Restaurar el beam visual del boton de contacto sin revertir el resto de optimizaciones de ruta critica mobile.

## Cambios incluidos

- `BorderBeam` vuelve a usar la implementacion Motion previa para animar `offsetDistance`
- el CTA `Contactar` vuelve a montar el beam como hijo directo del link, sin wrapper intermedio
- se retiran los keyframes CSS especificos de `border-beam`
- se actualizan referencia tecnica, decisions log y change-log

## Impacto en arquitectura / producto

Impacto acotado al CTA desktop del navbar y a documentacion. No cambia rutas, copy, backend, datos, SEO ni el resto de optimizaciones del pase PageSpeed.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- smoke local del navbar desktop

## Riesgos o arrastres

`BorderBeam` vuelve a arrastrar Motion en esta surface concreta, pero se prioriza fidelidad visual del CTA frente a una sustitucion CSS que no reproducia el resultado.

## Notas para presupuesto

Rollback selectivo de bug visual: mantiene la optimizacion general y restaura la primitive que tenia una dependencia justificada de Motion.
