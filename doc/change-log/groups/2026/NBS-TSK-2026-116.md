# Resumen

Grupo que recoge el aumento de tamaño de los nodos del timeline cuando se renderizan como imagen en lugar de icono.

## Objetivo del grupo

Mejorar la presencia visual de los assets ilustrados sin extender ese cambio a los nodos convencionales del sistema.

## Cambios incluidos

- nuevas constantes de tamaño para nodo ilustrado en desktop y mobile
- incremento de la caja visual del planeta
- sincronización de `sizes` con el nuevo tamaño

## Impacto en arquitectura / producto

No altera la estructura del timeline ni el comportamiento del SVG. Ajusta jerarquía visual de los nodos basados en imagen.

## Validacion agregada

- `npm run lint`
- `npm run build`

## Riesgos o arrastres

- un aumento adicional podría exigir reajuste de separación con la card o de placement horizontal
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Pulido visual localizado para reforzar la lectura de marca del nodo ilustrado.
