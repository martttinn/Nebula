# Resumen

Grupo que recoge la correccion del avance a tirones en la coreografia desktop de `projects-showcase` cuando el usuario hace scroll muy lentamente.

## Objetivo del grupo

Mantener la misma composicion visual y el mismo efecto de empuje entre proyectos, pero evitando que el progreso se cuantice en saltos perceptibles con Lenis.

## Cambios incluidos

- se reduce el umbral de actualizacion de progreso de `projects-showcase`
- se valida el empuje de cards en produccion local con scroll lento simulado
- se preserva el snap programatico y la coreografia visual existente

## Impacto en arquitectura / producto

Impacto acotado al runtime frontend desktop de la seccion `Proyectos destacados`. No cambia copy, layout final, rutas, backend, SEO ni datos.

## Validacion agregada

- `npm run build`
- `npm run lint`
- `npm run typecheck`
- `npm run changes:sync`
- prueba local con Playwright sobre `next start` en `127.0.0.1:3100`

## Riesgos o arrastres

El umbral mas fino puede provocar mas renders durante el tramo activo de proyectos, pero el coste esta acotado a tres cards desktop y evita una regresion visible de suavidad.

## Notas para presupuesto

Bugfix de interaccion visual premium: ajuste quirurgico del umbral de progreso para eliminar tirones en scroll lento sin redisenar la seccion.
