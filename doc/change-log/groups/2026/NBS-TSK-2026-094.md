# Resumen

Grupo que recoge un nuevo ajuste del ritmo de coloreado palabra a palabra en la banda sticky de benefits.

## Objetivo del grupo

Hacer que cada palabra necesite más scroll real antes de alcanzar su color final, sin romper la secuencia sticky ni la salida de cada frase.

## Cambios incluidos

- ampliación del tramo de `progressRange` usado para el coloreado de cada frase
- retraso leve del inicio de salida para que el fade-out no se pise con el final del coloreado
- actualización de `DESIGN.md` para fijar la nueva exigencia de scroll

## Impacto en arquitectura / producto

La banda de benefits gana una lectura más pausada y más editorial, reforzando la sensación de progresión guiada por scroll.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si se sigue ralentizando más, podría sentirse demasiado pesada en trackpads cortos o viewport bajos

## Notas para presupuesto

Refinamiento de motion y timing en una sección pública clave.
