# Resumen

Grupo que recoge la desconexión temporal de Supabase del runtime público mientras Nebula Studios prioriza el trabajo de frontend.

## Objetivo del grupo

Mantener el bootstrap repo-safe de Supabase para fases posteriores, pero retirar su uso efectivo del runtime público para reducir fricción durante la iteración visual y estructural de la web.

## Cambios incluidos

- retirada del puente runtime de Supabase para que no exista `proxy` o `middleware` activo
- corrección del script `lint` para compatibilidad real con `next@16`
- actualización de `README.md` y `.env.example` para dejar claro que Supabase sigue aparcado
- actualización de reglas de contexto, referencia técnica y decisiones duraderas
- registro de la entrega en `doc/change-log/`

## Impacto en arquitectura / producto

La web pública vuelve a operar como frontend puro, sin refresco de sesión ni dependencia efectiva de variables de entorno de Supabase, mientras se conserva intacto el bootstrap necesario para reactivar backend más adelante.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- cuando se reactive Supabase habrá que volver a montar el puente runtime y revisar el contrato de envs
- el bootstrap queda presente pero inactivo, así que la documentación debe seguir tratándolo como base futura y no como backend real

## Notas para presupuesto

Trabajo de saneamiento operativo y foco de alcance: no añade funcionalidades visibles, pero reduce fricción para avanzar rápido en frontend sin destruir la preparación backend ya hecha.
