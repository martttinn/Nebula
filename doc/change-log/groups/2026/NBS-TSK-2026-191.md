# Resumen

Grupo que recoge la unificacion responsive del stack sticky de `testimonials` para que mobile y tablet usen el mismo apilado progresivo que desktop.

## Objetivo del grupo

Eliminar la lista vertical movil independiente y mantener una experiencia de prueba social consistente en todos los breakpoints, ajustando solo footprint, escala y labio del stack para viewports compactos.

## Cambios incluidos

- Sustitucion del render movil separado por un unico stage sticky responsive.
- Parametrizacion del stack con valores compactos para `<1024px`.
- Ajuste de la jerarquia interna de las cards para conservar legibilidad dentro del viewport movil.
- Actualizacion de `DESIGN.md` con el canon responsive de `testimonials`.

## Impacto en arquitectura / producto

Impacto localizado en la seccion publica `testimonials` de la home. Mejora continuidad visual y percepcion premium sin cambiar contenido, rutas, datos, CTA, backend ni dependencias.

## Validacion agregada

- Check rojo/verde de codigo para verificar que mobile ya no usa lista simple.
- `npm run lint`
- `npm run typecheck`
- `npm run build`
- Verificacion visual mobile/desktop con Playwright
- `npm run changes:sync`

## Riesgos o arrastres

El stage sticky se activa ahora tambien en mobile/tablet. El riesgo principal es altura disponible en pantallas pequenas; se mitiga con tipografia, padding, escala y labio compactos.

## Notas para presupuesto

Mejora visible de UX/motion responsive en una seccion publica clave de prueba social.
