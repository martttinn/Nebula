# Resumen

Grupo que recoge una primera corrección acotada de hallazgos de React Doctor sobre performance, dead code real y maintainability en runtime público y tooling local.

## Objetivo del grupo

Reducir avisos accionables de bajo riesgo sin convertir la auditoría en un refactor amplio ni tocar hallazgos con falsos positivos claros o entrypoints externos.

## Cambios incluidos

- batch de escrituras imperativas de estilo en preloader y canvas del hero
- reducción del blur animado principal del preloader
- normalización de pares `w-N h-N` a `size-N` donde Tailwind lo soporta
- retirada de exports públicos no consumidos y de un primitive `slot` sin imports
- reutilización de `Intl.Segmenter` en `SplitText`
- optimización local del validador/generador de change-log con `Map`/`Set`
- mantenimiento de React Doctor como herramienta bajo demanda sin instalar dependencias

## Impacto en arquitectura / producto

No cambia la arquitectura ni la narrativa pública. El impacto es de rendimiento y mantenimiento: menos escrituras DOM secuenciales, clases Tailwind más compactas, menos API pública accidental y menos ruido de auditoría.

## Validacion agregada

- `npx react-doctor@latest . --verbose` (`134` issues antes del cambio compartido por Martín; `81` issues y `87/100` tras la limpieza)
- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm run changes:sync`

## Riesgos o arrastres

Quedan hallazgos no tratados deliberadamente: dead code con falsos positivos de Supabase/Remotion, migración amplia a LazyMotion, headings de Remotion/UI y refactors de estado que requieren más cuidado.

## Notas para presupuesto

Primera pasada de saneamiento técnico derivada de diagnóstico automatizado React Doctor.
