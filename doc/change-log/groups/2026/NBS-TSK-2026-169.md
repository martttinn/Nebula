# Resumen

Grupo que recoge la restauracion de `HeroParticles` como capa atmosferica de continuidad en la home publica.

## Objetivo del grupo

Recuperar la presencia visual de particulas en servicios, proceso y testimonials sin revertir los demas ajustes de performance aplicados en el hero, el preloader y el CTA final.

## Cambios incluidos

- restauracion de `HeroParticles` en `services-carousel`
- restauracion de `HeroParticles` en `how-we-work`
- restauracion de `HeroParticles` en `testimonials`
- actualizacion de `DESIGN.md`, decisions log y referencia tecnica para reflejar el runtime actual

## Impacto en arquitectura / producto

Impacto visual y de runtime frontend. No cambia copy, rutas, SEO, backend, auth ni datos. El cambio recupera continuidad atmosferica entre secciones a costa de aceptar de nuevo un riesgo de performance/CLS que ya habia sido medido como trade-off.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm run changes:sync`

## Riesgos o arrastres

La restauracion puede reintroducir parte del CLS de sesion observado antes de retirar la capa. Si Vercel Speed Insights vuelve a penalizar `CLS`, esta decision debe considerarse un trade-off visual consciente.

## Notas para presupuesto

Cambio acotado de restauracion visual con trazabilidad documental. El valor es perceptivo/branding; el coste potencial es rendimiento de campo en sesiones con scroll profundo.
