# Resumen

Grupo que recoge la integración del footer público de Nebula como cierre de marca + navegación, con links legales visibles pero desactivados y catálogo compartido con el navbar.

## Objetivo del grupo

Cerrar la home con una surface sobria de marca y navegación que reutilice la información pública existente, añada los destinos legales pendientes y evite drift entre navbar y footer.

## Cambios incluidos

- creación de `components/layout/footer.tsx`
- extracción de `components/layout/brand-lockup.tsx` como primitive compartida
- creación de `data/navigation.ts` como fuente de verdad para links públicos y legales
- actualización de `navbar.tsx` y `navbar-staggered-menu.tsx` para consumir ese catálogo compartido
- integración del footer al final de `app/page.tsx`
- actualización de `DESIGN.md`, referencias, `decisions-log` y `change-log`

## Impacto en arquitectura / producto

La home gana un cierre global explícito y más completo, mientras navbar y footer dejan de duplicar enlaces a mano. El catálogo compartido simplifica evolución futura de navegación y hace visible el estado real de los destinos legales sin publicarlos antes de tiempo.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm run changes:sync`

## Riesgos o arrastres

- `Precios`, `Nosotros` y `Blog` siguen siendo anclas futuras ya existentes en navbar; el footer hereda ese mismo estado para no divergir
- los links legales quedan visibles pero no operativos hasta que existan sus rutas reales

## Notas para presupuesto

Trabajo visible y estructural a la vez: diseño e integración del footer, normalización de navegación compartida entre shells globales y explicitación del estado real de los destinos legales.
