# Resumen

Grupo que recoge la asignación de un icono 3D específico a cada servicio dentro del carrusel de la home.

## Objetivo del grupo

Diferenciar visualmente cada card usando el set de assets disponible en `public/3d-Icons` en vez de reutilizar un único icono para todo el carrusel.

## Cambios incluidos

- ampliación de `data/services.ts` para incluir `iconSrc` por servicio
- mapeo de `iphone`, `webPanel`, `grafico` y `lupa` a móvil, web, evolución y consultoría/digitalización
- actualización de `components/home/services-carousel.tsx` para consumir el icono desde el catálogo
- actualización de `DESIGN.md`, referencia técnica y `decisions-log`

## Impacto en arquitectura / producto

La sección conserva su composición y mecánica, pero gana diferenciación visual por servicio sin añadir nueva lógica de render ni nuevos componentes.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- el set actual cubre exactamente cuatro servicios; si el catálogo crece, hará falta ampliar `public/3d-Icons` o decidir reutilizaciones explícitas

## Notas para presupuesto

Refinamiento visual del catálogo de servicios mediante mapeo de assets 3D existentes.
