# Resumen

Grupo que recoge la recomposición del hero para usar GridScan como fondo real a ancho completo y altura de viewport.

## Objetivo del grupo

Convertir el efecto GridScan en el fondo real del hero, de forma que la primera sección ocupe todo el ancho visible y una altura equivalente al viewport completo.

## Cambios incluidos

- recomposición de `app/page.tsx` para sacar el hero de un contenedor acotado
- paso a layout full-bleed con `GridScan` como fondo visual principal
- ajuste de overlays, jerarquía tipográfica y paneles de soporte sobre el nuevo fondo
- registro de la entrega en `doc/change-log/`

## Impacto en arquitectura / producto

La home gana una primera pantalla mucho más coherente con la intención visual de Nebula: el efecto ya no se percibe como demo embebida, sino como atmósfera principal de marca.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- el hero pasa a depender aún más del equilibrio entre efecto visual y legibilidad, así que futuros cambios de copy o CTA deben revisarse visualmente
- la dirección full-bleed exige cuidado extra si luego se introducen navegación fija o banners superiores

## Notas para presupuesto

Trabajo de frontend premium orientado a composición y dirección de arte: no cambia lógica de negocio, pero sí la percepción inicial del producto y la claridad del hero.
