# Resumen

Grupo que recoge la incorporación de una capa atmosférica de partículas lilas suaves en el hero de la home.

## Objetivo del grupo

Añadir profundidad y movimiento muy sutil al hero sin introducir un nuevo runtime cliente ni competir con `GridScan` o con el copy principal.

## Cambios incluidos

- creación de `components/home/hero-particles.tsx`
- creación de `components/home/hero-particles.module.css`
- integración de la capa de partículas en `Hero`
- definición del guardrail visual en `DESIGN.md`
- alineación de la referencia técnica y del `change-log`

## Impacto en arquitectura / producto

El hero gana una capa decorativa adicional de baja intensidad, implementada como componente server-side y animación CSS, sin aumentar la complejidad del shell principal.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- si se incrementa demasiado la densidad o el brillo, la capa puede degradarse hacia ruido visual y competir con la lectura del hero

## Notas para presupuesto

Refinamiento atmosférico del hero con motion sutil y coste técnico contenido.
