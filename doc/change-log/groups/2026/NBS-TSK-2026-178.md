# Resumen

Grupo que recoge la retirada del glow restante en el hover de la información de contacto del footer.

## Objetivo del grupo

Eliminar la sombra luminosa del icono de email y teléfono manteniendo el feedback de borde, fondo y color.

## Cambios incluidos

- `components/layout/footer.tsx` elimina las clases de sombra `group-hover/contact` y `group-focus-visible/contact` del wrapper del icono.
- `DESIGN.md` explicita que la información de contacto del footer no debe usar glow ni sombra luminosa.

## Impacto en arquitectura / producto

El cambio es puramente visual. No altera enlaces, datos de contacto, layout, navegación, rutas ni backend.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- smoke visual hover desktop en `http://localhost:3001`
- `npm run changes:sync`

## Riesgos o arrastres

La microinteracción queda más sobria. Si se quiere reducirla todavía más, el siguiente ajuste sería quitar la escala o el fondo activo, no el glow.

## Notas para presupuesto

Ajuste menor de microinteracción visual del footer.
