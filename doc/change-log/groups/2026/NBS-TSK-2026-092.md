# Resumen

Grupo que recoge la integración de `BorderGlow` en las cards del carrusel de servicios y la adaptación necesaria para que el efecto sea realmente visible en runtime.

## Objetivo del grupo

Aplicar un glow perimetral premium a las cards de servicios sin alterar su geometría base ni romper la legibilidad del contenido.

## Cambios incluidos

- importación de la receta `@react-bits/BorderGlow-TS-CSS`
- adaptación de la primitive a `CSS Modules` para encajar con Next App Router
- integración de `BorderGlow` dentro de `ServiceCard`
- refuerzo del halo perimetral y simplificación del efecto para hacerlo legible sobre fondos oscuros en Chrome
- validación del comportamiento mediante prueba real de hover sobre `localhost:3000`

## Impacto en arquitectura / producto

La home gana una nueva primitive visual reusable y las cards de servicios obtienen un acento interactivo sutil, visible y coherente con el lenguaje dark-tech de Nebula.

## Validacion agregada

- prueba real en navegador local con hover automatizado sobre la card `Desarrollo web`
- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si el sistema visual usa este glow en otras superficies habrá que recalibrar intensidad según fondo, densidad y contraste local

## Notas para presupuesto

Refinamiento visual e integración de primitive interactiva en sección pública clave.
