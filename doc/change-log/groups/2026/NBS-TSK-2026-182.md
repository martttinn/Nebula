# Resumen

Grupo que recoge la migracion completa de las animaciones declarativas del runtime publico desde `framer-motion` hacia Motion for React via `motion/react`.

## Objetivo del grupo

Eliminar el acoplamiento directo del proyecto a `framer-motion`, mantener la API declarativa de motion ya usada en la UI y dejar el stack documentado sobre la dependencia vigente `motion`.

## Cambios incluidos

- Imports de componentes y hooks migrados de `framer-motion` a `motion/react`.
- Dependencia directa `framer-motion` retirada de `package.json`.
- `package-lock.json` regenerado sin `node_modules/framer-motion` root.
- Documentacion viva del stack y reglas locales actualizadas a Motion for React.
- Referencias operativas internas normalizadas para no recomendar `framer-motion` como stack activo.

## Impacto en arquitectura / producto

Impacto estructural y de mantenimiento: conserva la experiencia visual existente, reduce duplicidad de librerias de animacion declarativa y alinea el codigo con la API actual recomendada por Motion.

## Validacion agregada

- `rg "from ['\"]framer-motion['\"]|framer-motion|Framer Motion"`
- `npm ls framer-motion motion --depth=2`
- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm run changes:sync`

## Riesgos o arrastres

`framer-motion` puede seguir apareciendo en `package-lock.json` como dependencia transitiva interna de `motion`; no debe tratarse como dependencia directa del proyecto mientras `package.json` no lo declare ni existan imports runtime propios.

## Notas para presupuesto

Mantenimiento tecnico transversal con impacto principalmente estructural: reduce drift de dependencias, simplifica el stack de animacion y mantiene trazabilidad documental.
