# Resumen

Grupo que recoge la retirada de las fachadas de compatibilidad del hero y el ajuste de imports internos para que la home y sus secciones dependientes consuman ya la carpeta canónica `components/home/hero/`.

## Objetivo del grupo

Cerrar la migración arquitectónica de la home para que todas sus familias principales resuelvan directamente sus directorios canónicos y no wrappers heredados.

## Cambios incluidos

- retirada de `components/home/hero.tsx`
- retirada de `components/home/hero-lead.tsx`
- retirada de `components/home/hero-particles.tsx`
- retirada de `components/home/hero-particles.module.css`
- retirada de `components/home/grid-scan.tsx`
- retirada de `components/home/grid-scan-shell.tsx`
- retirada de `components/home/grid-scan-fallback.tsx`
- actualización de imports de `HeroParticles` en `services-carousel` y `how-we-work` hacia la ruta canónica

## Impacto en arquitectura / producto

La home mantiene los mismos specifiers de alto nivel, pero la resolución real ya no pasa por archivos intermedios. El hero queda como familia de carpeta y las demás secciones dependen ya de esa fuente de verdad.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si el editor tenía tabs abiertas de rutas antiguas del hero, habrá que reabrir los archivos desde la carpeta nueva
- las referencias históricas del change-log siguen apuntando a rutas antiguas porque describen estados pasados del runtime
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Paso final de consolidación arquitectónica de la home pública, orientado a cerrar la transición de compatibilidad y a dejar un árbol de componentes coherente para evolución futura.
