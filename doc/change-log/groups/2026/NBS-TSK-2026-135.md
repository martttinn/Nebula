# Resumen

Grupo que recoge el ajuste del progreso de dibujo del path SVG de `How we work` para que complete su recorrido antes de que la sección desaparezca casi por completo del viewport.

## Objetivo del grupo

Mantener la lectura del recorrido visual dentro del tramo útil de scroll de la sección, evitando que el último tramo del path se termine de pintar cuando el usuario ya está entrando en el bloque siguiente.

## Cambios incluidos

- introducción de una constante para acotar el punto de finalización del dibujo del path
- compresión del rango de progreso usado por `strokeDashoffset` en desktop

## Impacto en arquitectura / producto

No cambia la geometría ni la distribución de nodos y cards. Solo adelanta el completado visual del path dentro del mismo scroll de la sección.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si en futuras iteraciones cambian mucho los `DESKTOP_STEP_STOPS` o la altura efectiva de la sección, puede convenir recalibrar también este ratio

## Notas para presupuesto

Ajuste fino de narrativa scroll-driven para alinear mejor la animación del path con el tiempo real de lectura de la sección.
