# Resumen

Grupo que recoge la extraccion del carrusel de servicios de `nebula-legacy` y su integracion retemada dentro de la home actual de Nebula Studios.

## Objetivo del grupo

Recuperar una surface compleja ya validada visualmente en el proyecto legado para aterrizar los servicios publicos del estudio en la web actual, sin arrastrar su sistema visual anterior ni sus textos antiguos.

## Cambios incluidos

- creacion de `data/services.ts`
- creacion de `components/home/services-carousel.tsx`
- integracion de la seccion en `app/page.tsx`
- actualizacion de `DESIGN.md`, referencia tecnica y `decisions-log`

## Impacto en arquitectura / producto

La home gana una tercera seccion publica real y el repo vuelve a reutilizar una primitive de alto impacto procedente de `nebula-legacy`, ya adaptada al dominio y al lenguaje visual vigentes.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- no se ha hecho todavia revision visual automatizada del carrusel en navegador dentro de esta sesion
- `npm run changes:sync` sigue sin poder ejecutarse porque ese tooling no esta expuesto en `package.json`

## Notas para presupuesto

Entrega visible de frontend con extraccion de componente legado, adaptacion visual al sistema actual e integracion real en la home publica.
