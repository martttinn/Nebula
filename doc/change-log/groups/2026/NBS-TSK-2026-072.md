# Resumen

Grupo que recoge la correccion del corte visible entre la banda de propuesta de valor y la seccion de servicios.

## Objetivo del grupo

Eliminar la sensacion de salto entre ambas superficies haciendo que servicios continúe visualmente donde benefits termina.

## Cambios incluidos

- retirada del cierre uniforme en `Navy` al final de la banda de benefits
- reajuste del arranque de servicios para que parta ya desde `Void` con brillo muy contenido
- unificacion del fondo de servicios en una constante compartida entre desktop y movil
- actualizacion del contrato visual de `value-proposition-band` y `services-carousel` en `DESIGN.md`

## Impacto en arquitectura / producto

La mecanica del carrusel no cambia, pero la transicion entre secciones deja de mostrar un corte duro de fondo.

## Validacion agregada

- `npm run lint`
- `npm run build`

## Riesgos o arrastres

- el repo no expone `npm run design:lint`, asi que no se ha podido ejecutar esa validacion documental
- no se ha hecho revision visual automatizada en navegador en esta sesion

## Notas para presupuesto

Refinamiento visual de continuidad entre superficies publicas contiguas con ajuste de dos primitives documentadas.
