# Resumen

Grupo que recoge la corrección tipográfica de `SplitText` para impedir que el heading del hero rompa palabras por letras.

## Objetivo del grupo

Mantener el reveal por caracteres sin sacrificar la calidad del wrap del titular.

## Cambios incluidos

- agrupación de caracteres por palabra
- wrappers `nowrap` para cada palabra animada
- preservación de espacios como puntos válidos de salto

## Impacto en arquitectura / producto

La primitive reusable mejora su comportamiento en titulares multi-línea sin exigir cambios en el hero ni en otros consumidores.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- no hay arrastres funcionales esperados; solo conviene revisar visualmente el hero en tablet y móvil

## Notas para presupuesto

Corrección menor de composición tipográfica.
