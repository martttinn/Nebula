# Resumen

Grupo que recoge la recalibración del layout interno de las cards del carrusel de servicios después de aumentar su tamaño.

## Objetivo del grupo

Evitar que el CTA y el copy queden demasiado bajos o recortados al crecer las cards, ajustando la composición interna como un sistema completo.

## Cambios incluidos

- aumento de la altura útil de `ServiceCard`
- recalibración de título, icono central y columna media para liberar espacio real al footer
- más padding inferior y footer más estable para que el botón quede siempre visible
- ligera elevación del bloque de cards dentro del stage
- refuerzo del truncado temprano de la descripción para blindar el layout
- actualización de `DESIGN.md`
- aumento posterior de la huella del CTA `Ver más` para darle más presencia sin romper la jerarquía de la card

## Impacto en arquitectura / producto

La sección mantiene el mismo patrón sticky y el mismo catálogo, pero mejora legibilidad y estabilidad visual tras el aumento de escala.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si se sigue creciendo en tamaño, habrá que revisar también el runway y el framing general del stage desktop

## Notas para presupuesto

Refinamiento de layout interno y proporción visual en una sección pública clave.
