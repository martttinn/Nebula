# Resumen

Grupo que recoge la retirada del link `Blog` del navbar público de Nebula Studios.

## Objetivo del grupo

Eliminar `Blog` de la navegación principal desktop y móvil sin borrar el catálogo ampliado que sigue usando el footer.

## Cambios incluidos

- `data/navigation.ts` añade `PUBLIC_NAVBAR_LINKS` como set específico para el navbar.
- `components/layout/navbar.tsx` usa `PUBLIC_NAVBAR_LINKS` como valor por defecto.
- `PUBLIC_NAV_LINKS` se mantiene como set ampliado para el footer.
- `DESIGN.md` y `doc/reference/technical-reference.md` documentan la separación de catálogos.

## Impacto en arquitectura / producto

El navbar deja de mostrar un destino de blog que todavía no tiene superficie pública real. El footer conserva su estructura y puede seguir mostrando enlaces de estudio sin modificar el layout pedido previamente.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- smoke visual desktop en `http://localhost:3001`
- smoke visual móvil en `http://localhost:3001`
- `npm run changes:sync`

## Riesgos o arrastres

Existe ahora una distinción explícita entre links activos del navbar y links ampliados del footer. Esta separación evita borrar información del footer por accidente, pero requiere elegir el catálogo correcto en futuras superficies.

## Notas para presupuesto

Ajuste pequeño de navegación pública con mínima separación de datos para respetar el alcance exacto del cambio.
