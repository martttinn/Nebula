# Resumen

Grupo que recoge el pase de reduccion de ruta critica mobile aplicado sobre la home publica tras las sugerencias PageSpeed compartidas el `2026-05-24`.

## Objetivo del grupo

Reducir CSS bloqueante y coste de microanimaciones del primer viewport sin cambiar el resultado visual ni retirar el preloader de marca.

## Cambios incluidos

- `StaggeredMenu.css` deja de formar parte del CSS critico inicial y viaja con el chunk diferido del menu responsive
- el fallback inicial del menu movil queda estilado con Tailwind para no depender del stylesheet diferido
- `Button`, `BorderBeam`, `Preloader`, `SplitText`, `NebulaLogoAnimated`, `HeroLead` y el shell del navbar sustituyen usos above-the-fold de `motion/react` por CSS transitions/keyframes o estado React minimo
- se anade `lib/use-prefers-reduced-motion.ts` para respetar `prefers-reduced-motion` sin importar el runtime de Motion en superficies ligeras
- se documenta que el aviso PageSpeed de polyfills legacy procede del runtime cliente de Next y no se modifica con alias interna sin impacto medible

## Impacto en arquitectura / producto

Impacto limitado a frontend runtime, primitives UI y documentacion. No cambia copy, layout, identidad visual, rutas, SEO, backend, auth, datos ni dependencias.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `git diff --check`
- inspeccion de stylesheets y scripts emitidos en `.next/server/app/index.html`
- smoke visual local con Playwright sobre mobile y desktop

## Riesgos o arrastres

El preloader se mantiene por criterio visual, por lo que el retraso de renderizado del LCP solo bajara de forma limitada mientras el overlay siga siendo parte de la experiencia. La eliminacion completa del chunk inicial de Motion requiere refactorizar secciones below-the-fold que todavia usan scroll-driven animation y queda fuera de este pase para evitar drift visual.

## Rollback

El cambio se implementa en la rama `codex/mobile-critical-path-perf`, creada desde `main` en `5182b07`. Para volver al estado previo basta con cambiar a `main` o descartar esta rama antes de mergear.

## Notas para presupuesto

Trabajo de performance conservador: reduce CSS y dependencias de arranque sin rediseñar ni retirar piezas de marca.
