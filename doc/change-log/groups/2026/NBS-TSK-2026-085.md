# Resumen

Grupo que recoge el aumento de tamaño de las cards del carrusel de servicios y la apertura adicional del arco desktop para separarlas más.

## Objetivo del grupo

Hacer que las cards ganen presencia visual y respiren más entre sí, manteniendo la alineación curvada del carrusel.

## Cambios incluidos

- ampliación del tamaño base de `ServiceCard`
- incremento del radio y del span angular del arco desktop para separar más las cards
- ajuste menor del layout interior de la card y del stage móvil para acomodar la nueva geometría
- actualización de `DESIGN.md`

## Impacto en arquitectura / producto

La sección mantiene su misma mecánica sticky, pero cambia su footprint visual y su densidad en desktop hacia una lectura más abierta y más premium.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si el arco se abre todavía más en futuras iteraciones, algunas cards laterales podrían empezar a recortarse demasiado en viewports medianos

## Notas para presupuesto

Refinamiento visual de layout y geometría de una sección pública clave.
