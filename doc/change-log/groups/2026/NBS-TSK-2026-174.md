# Resumen

Grupo que recoge la mejora de microinteracción para los enlaces de contacto del footer.

## Objetivo del grupo

Dar feedback visual claro al email y teléfono del footer sin cambiar el layout ni convertir la zona de contacto en una card.

## Cambios incluidos

- `components/layout/footer.tsx` añade hover/focus de grupo a email y teléfono.
- Los iconos ganan glow, borde activo y fondo sutil en interacción.
- El texto añade subrayado fino progresivo.
- `DESIGN.md` documenta el criterio canónico para esta microinteracción.

## Impacto en arquitectura / producto

El cambio es puramente visual y queda acotado al footer. No altera enlaces, rutas, navegación, datos de contacto ni backend.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- smoke visual hover desktop en `http://localhost:3001`
- `npm run changes:sync`

## Riesgos o arrastres

El hover usa transform visual y sombras, pero no cambia el espacio ocupado. El estado táctil móvil conserva el link clicable sin depender del hover.

## Notas para presupuesto

Entrega menor de pulido visual con impacto en affordance y percepción de acabado del footer.
