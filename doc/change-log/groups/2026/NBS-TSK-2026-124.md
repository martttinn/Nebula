# Resumen

Grupo que recoge la reorganización interna de la sección `How we work` para pasar de un archivo monolítico a una carpeta con piezas acotadas por responsabilidad.

## Objetivo del grupo

Mejorar mantenibilidad, legibilidad y velocidad de iteración sobre la sección de proceso sin alterar su API pública ni su comportamiento en runtime.

## Cambios incluidos

- sustitución de `components/home/how-we-work.tsx` por una carpeta `components/home/how-we-work/`
- separación de configuración, tipos, geometría SVG, utilidades y primitives visuales
- actualización de la referencia técnica para reflejar la nueva estructura

## Impacto en arquitectura / producto

Cambio interno de frontend. No modifica copy, layout, motion ni contratos de consumo desde `app/page.tsx`.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- la validación final sigue requiriendo comprobación visual manual de la sección en navegador
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Refactorización estructural de mantenimiento con retorno en claridad operativa y menor coste de futuros cambios sobre la sección de proceso.
