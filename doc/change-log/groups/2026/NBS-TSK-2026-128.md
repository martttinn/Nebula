# Resumen

Grupo que recoge la retirada de las fachadas de compatibilidad de `value-proposition` para que la home resuelva directamente la carpeta canónica de la sección.

## Objetivo del grupo

Avanzar la migración desde un estado de compatibilidad segura a un consumo directo de la nueva arquitectura sin alterar el comportamiento del runtime.

## Cambios incluidos

- retirada de `components/home/value-proposition-section.tsx`
- retirada de `components/home/value-proposition-statements.tsx`
- retirada de `components/home/value-proposition-ornaments.module.css`

## Impacto en arquitectura / producto

La home sigue importando el mismo specifier, pero ahora este resuelve directamente el `index.tsx` del directorio `components/home/value-proposition-section/`.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- si el editor tenía abiertas tabs antiguas de compatibilidad, esas rutas dejarán de existir y habrá que reabrir los archivos desde la carpeta nueva
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Paso corto de consolidación arquitectónica orientado a eliminar deuda de transición después de la modularización previa de la sección.
