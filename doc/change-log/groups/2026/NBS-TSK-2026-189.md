# Resumen

Grupo que recoge la incorporacion de snap programatico en la seccion desktop de proyectos destacados.

## Objetivo del grupo

Evitar que el usuario pueda dejar el scroll detenido en puntos intermedios de la coreografia entre proyectos, especialmente durante los empujes laterales entre paneles full-screen.

## Cambios incluidos

- Ajuste de la timeline de `projects-showcase` para que cada panel tenga un final de reveal explicito.
- Activacion de cuatro puntos de reposo desktop: primera card compacta, primera card full-screen, segunda card full-screen y tercera card full-screen.
- Documentacion del comportamiento canonico en `DESIGN.md`.

## Impacto en arquitectura / producto

Cambio local en una isla cliente de la home. Mejora la legibilidad del scroll storytelling sin alterar contenido, datos, rutas, CTA ni la degradacion movil/tablet.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- Revision visual desktop/mobile de `projects-showcase`
- `npm run changes:sync`

## Riesgos o arrastres

El snap depende de Lenis en desktop. Se mantiene desactivado en mobile/tablet y con `prefers-reduced-motion` para no endurecer el scroll natural ni la accesibilidad.

## Notas para presupuesto

Feature visible de UX/motion sobre una seccion publica de portfolio, con ajuste de runtime, canon visual y registro documental.
