# Resumen

Grupo que recoge la normalización del hover por defecto de la variante `secondary` del sistema de botones y la posterior eliminación de overrides locales que seguían rebajando su opacidad.

## Objetivo del grupo

Hacer que cualquier consumo de `Button` en variante `secondary` herede el mismo patrón de hover: escala `1.05`, fondo blanco y texto negro.

## Cambios incluidos

- actualización de la variante `secondary` en `components/animate-ui/components/buttons/button.tsx`
- especialización del `resolvedHoverScale` por defecto para que `secondary` use `1.05`
- alineación de consumos concretos del hero y del carrusel de servicios para que el hover use blanco al `100%`
- actualización de `DESIGN.md`

## Impacto en arquitectura / producto

La primitive pública de botones gana un comportamiento más consistente y los consumos dejan de depender de overrides repetidos para alcanzar el estado hover esperado.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- cualquier consumo existente de `secondary` que esperase un hover más tenue heredará ahora un estado más contrastado y una escala mayor

## Notas para presupuesto

Normalización de primitive reusable con impacto transversal en la UI.
