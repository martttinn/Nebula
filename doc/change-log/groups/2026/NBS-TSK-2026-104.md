# Resumen

Grupo que recoge el ajuste de curvatura en los tramos extremos del timeline desktop de `How we work`.

## Objetivo del grupo

Evitar que la linea haga una curva exagerada antes del primer nodo y despues del ultimo, manteniendo intacta la personalidad serpenteante del recorrido central.

## Cambios incluidos

- ajuste de `buildSmoothPath` para que los segmentos extremos usen su propio delta vertical como referencia de tangencia

## Impacto en arquitectura / producto

No cambia la estructura de la home ni el layout del timeline. Refina solo la calidad geometrica del path SVG en una superficie publica.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- la validacion automatizada confirma compilacion y tipado, pero la aprobacion estetica final sigue requiriendo revision visual en navegador
- `DEBUG_BORDERS` sigue activo y puede enmascarar parcialmente la lectura final del acabado si no se desactiva despues
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Pulido visual pequeño pero visible sobre una seccion editorial protagonista de la home publica.
