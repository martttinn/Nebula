# Resumen

Grupo que integra en Nebula Studios el tooling local de `change-log` ya probado en Future Nova App, adaptado al historial y a las reglas reales de este repo.

## Objetivo del grupo

Convertir el `change-log` documentado en un subsistema ejecutable: crear entradas y grupos por script, validar consistencia e indexar el histórico sin depender de trabajo manual frágil.

## Cambios incluidos

- revisión del tooling existente en `/Users/martin/Documents/Nebula Studios/Apps/FutureNova-App/scripts`
- port de `change-log-lib.js`, `create-change-entry.js`, `create-change-group.js`, `validate-change-log.js` y `build-change-log-index.js`
- adaptación del tooling al prefijo `NBS`, a `doc/change-log/config.json` y al histórico real del repo
- compatibilidad deliberada con entradas legacy y degradación de IDs duplicados históricos a warnings en vez de bloqueo duro
- reparación del desajuste activo entre la entrada `152` y su grupo real
- sincronización de la referencia técnica y del contexto operativo para dejar de afirmar que `changes:*` no existe

## Impacto en arquitectura / producto

Impacto directo en tooling y workflows repo-locales. El `change-log` deja de ser una convención puramente documental y pasa a ser un subsistema verificable, generable e indexable.

## Validacion agregada

- `npm run changes:validate`
- `npm run changes:index`
- `npm run changes:sync`
- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- el validador no migra automáticamente el histórico antiguo; lo tolera y lo hace visible sin forzar una reescritura masiva
- siguen existiendo IDs históricos duplicados en el árbol, pero ahora se reportan como warnings para permitir sincronización y limpieza posterior
- `design:lint` sigue documentado en algunas reglas como expectativa, pero no existe todavía en `package.json`

## Notas para presupuesto

Entrega interna de hardening y DX: no cambia el runtime público, pero reduce drift documental, acelera el trabajo futuro y baja el coste operativo de mantener el `change-log`.
