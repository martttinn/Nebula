# Resumen

Grupo que recoge la reactivacion del modo debug visual en la seccion `How we work`.

## Objetivo del grupo

Volver a exponer los outlines de depuracion en runtime para revisar el layout de la seccion y su timeline serpenteante.

## Cambios incluidos

- activacion del flag local `DEBUG_BORDERS` en `components/home/how-we-work.tsx`

## Impacto en arquitectura / producto

No cambia estructura, contratos ni comportamiento funcional. Solo expone una capa temporal de diagnostico visual sobre una seccion publica existente.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- los bordes debug quedan visibles en runtime hasta que el flag vuelva a desactivarse
- `npm run changes:sync` sigue sin poder ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Ajuste puntual de inspeccion visual para iterar sobre layout y alineacion.
