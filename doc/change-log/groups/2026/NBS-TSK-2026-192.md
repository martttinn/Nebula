# Resumen

Grupo que recoge la optimizacion conservadora aplicada tras la auditoria PageSpeed Insights mobile de `somosnebula.com`.

## Objetivo del grupo

Mejorar el coste de JavaScript, hidratacion y main thread sin alterar el resultado visual de la home publica.

## Cambios incluidos

- `Preloader` con progreso animado sin intervalo de estado React
- carga diferida por intencion/idle del `StaggeredMenu` responsive y GSAP
- soporte `initialOpen` para preservar la apertura inmediata del menu tras cargar el chunk diferido
- sustitucion del consumo cliente de `tech-stack-icons` por un subconjunto SVG local de los iconos realmente usados
- actualizacion de decisions log, referencia tecnica y change-log

## Impacto en arquitectura / producto

Impacto frontend runtime y documentacion. No cambia copy, SEO, rutas, backend, auth, datos ni identidad visual. La home conserva la misma composicion y direccion visual, pero reduce trabajo no critico en el arranque.

## Validacion agregada

- PageSpeed Insights mobile oficial consultado el `2026-05-22`
- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npx react-doctor@latest --offline --full --no-lint --json --json-compact`
- inspeccion de chunks `.next/static/chunks` tras build
- busqueda post-build sin referencias a `tech-stack-icons` ni `Icon with name`
- Playwright CLI sobre `next start --port 3100` para hero desktop, menu mobile y badges de proyectos

## Riesgos o arrastres

La mejora de PageSpeed solo queda confirmada tras desplegar y medir de nuevo el dominio oficial. El primer tap del menu responsive puede esperar a que cargue el chunk diferido si ocurre antes del preload por idle; se mitiga cargando por foco, hover, touch o idle tardio y abriendo el menu con `initialOpen`.

`tech-stack-icons` queda instalado aunque ya no sea usado por el runtime publico; retirarlo del manifiesto puede abordarse como limpieza de dependencias separada.

## Notas para presupuesto

Trabajo de performance con impacto de producto: reduce friccion de primera impresion y deuda de bundle manteniendo el acabado visual actual, con trazabilidad documental para futuras iteraciones de la home.
