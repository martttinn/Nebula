# Resumen

Grupo que recoge la integración mínima y aislada de Remotion dentro del repo para abrir Studio y renderizar vídeos locales sin afectar al runtime público de Next.js.

## Objetivo del grupo

Habilitar un sistema local de vídeo preparado para crecer, pero manteniendo una frontera clara entre la web pública y el pipeline de render.

## Cambios incluidos

- instalación de `remotion`, `@remotion/cli` y `@remotion/google-fonts` como `devDependencies`
- nuevos scripts npm para Studio, render de vídeo y render de still
- creación del subsistema `remotion/` con composición inicial, tokens, fuentes y helpers de assets
- ajuste mínimo de ESLint para respetar el runtime CommonJS actual de `scripts/**/*.js` y poder mantener la validación en verde
- actualización de `.gitignore`, `decisions-log`, referencia técnica y `change-log`

## Impacto en arquitectura / producto

No hay impacto en producción ni en el routing público. El repo gana una segunda superficie técnica, solo de desarrollo, para generación de vídeo local con branding Nebula.

## Validacion agregada

- `npm run video:render -- NebulaShowcase renders/nebula-showcase.mp4`
- `npm run video:still -- NebulaShowcase renders/nebula-poster.png --frame 0`
- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- el subsistema de vídeo nace deliberadamente aislado y no reutiliza componentes web complejos; esa no-reutilización es intencional, no una carencia
- el siguiente umbral de complejidad todavía está por definir: presets de render, nombres de salida consistentes y posible separación futura a un workspace propio si el subsistema crece

## Notas para presupuesto

Entrega de infraestructura creativa y hardening arquitectónico: abre una nueva capacidad del repo con coste controlado hoy y deja preparada una vía de evolución a plantillas, composiciones múltiples o un workspace de vídeo dedicado.
