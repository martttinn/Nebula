# Resumen

Grupo que recoge la integración del efecto `DecryptedText` en el hero y la orquestación temporal del copy principal.

## Objetivo del grupo

Hacer que el hero entre con una jerarquía temporal clara: heading primero, subheading después y botones al final, sin competir con la salida del preloader.

## Cambios incluidos

- nueva primitive `components/ui/decrypted-text.tsx`
- nueva isla cliente `components/home/hero-lead.tsx`
- simplificación de `Hero` a shell + capa de contenido orquestada
- sincronización del arranque del hero con la salida del preloader
- actualización de `DESIGN.md`, decisiones y referencia técnica

## Impacto en arquitectura / producto

El hero gana una secuencia de motion más controlada y más alineada con la narrativa premium de la home, con una primitive reusable para futuros reveals tipográficos.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- el exceso de uso de `DecryptedText` en otras superficies degradaría el efecto; debe reservarse a momentos de jerarquía real

## Notas para presupuesto

Refactor frontend de alcance contenido con coordinación de motion entre varias primitives existentes y nuevas.
