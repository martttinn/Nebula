# Resumen

Grupo que recoge la correccion visual de la flecha de scroll inferior del hero tras la actualizacion del stack.

## Objetivo del grupo

Evitar que la linea vertical y el circulo de la flecha se separen durante la animacion, manteniendo la affordance como una unica pieza visual.

## Cambios incluidos

- Retirada de la linea vertical desposicionada de la cue inferior.
- Sustitucion de la variante Tailwind `motion-safe:*` por una clase CSS explicita con `prefers-reduced-motion`.

## Impacto en arquitectura / producto

Ajuste visual local en el hero de la home. No altera copy, rutas, datos, dependencias ni la secuencia de entrada del contenido.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- Playwright visual/DOM check hero scroll cue
- `npm run changes:sync`

## Riesgos o arrastres

La importacion `motion/react` ya formaba parte de la migracion de dependencias previa; este grupo solo cubre la correccion visual y de CSS de la cue.

## Notas para presupuesto

Correccion puntual de regresion visual post-upgrade en la zona above-the-fold.
