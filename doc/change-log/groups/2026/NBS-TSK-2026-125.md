# Resumen

Grupo que recoge la reorganización interna de la sección `services-carousel` para pasar de un único archivo a una carpeta modular con piezas separadas por responsabilidad.

## Objetivo del grupo

Mejorar claridad operativa y mantenibilidad de la sección de servicios sin alterar su API pública ni el comportamiento visual del carrusel sticky desktop y de la lista móvil.

## Cambios incluidos

- sustitución de `components/home/services-carousel.tsx` por la carpeta `components/home/services-carousel/`
- extracción de constantes, utilidades, geometría del arco, hook de scroll y primitives visuales
- actualización de la referencia técnica para reflejar la nueva estructura real

## Impacto en arquitectura / producto

Cambio interno de frontend. No modifica copy, catálogo de datos, composición visual ni contratos de consumo desde `app/page.tsx`.

## Validacion agregada

- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Riesgos o arrastres

- la validación visual final del arco desktop y de la lista móvil sigue siendo manual
- `npm run changes:sync` no puede ejecutarse porque los scripts `changes:*` no existen hoy en `package.json`

## Notas para presupuesto

Refactorización de mantenimiento con impacto positivo en velocidad de iteración y menor riesgo al tocar motion, layout o chrome visual de la sección de servicios.
