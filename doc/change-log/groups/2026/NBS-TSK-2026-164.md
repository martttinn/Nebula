# Resumen

Grupo que recoge la actualización del sistema de agentes para comprobar dependencias mencionadas contra el proyecto real antes de instalarlas o usarlas como stack.

## Objetivo del grupo

Evitar que paquetes citados en documentación, prompts o propuestas se traten como dependencias instaladas sin verificar manifiestos y lockfiles.

## Cambios incluidos

- nueva regla operativa para dependencias mencionadas
- actualización del flujo de arranque de tareas
- refuerzo de reglas de testing, release y documentación
- actualización del workflow y skill de auditoría de dependencias
- decisión duradera en `.agents/decisions-log.md`

## Impacto en arquitectura / producto

No cambia runtime. El impacto es operativo: los agentes deben distinguir dependencias instaladas, dependencias mencionadas/no instaladas y tooling bajo demanda antes de tocar código o manifiestos.

## Validacion agregada

- `npm run changes:sync`
- búsqueda `rg` sobre `dependencias mencionadas` y `mencionadas/no instaladas`

## Riesgos o arrastres

La regla añade una parada explícita cuando falten dependencias necesarias; puede ralentizar cambios rápidos, pero evita instalaciones implícitas y drift del stack.

## Notas para presupuesto

Endurecimiento documental del sistema de agentes y del protocolo de auditoría de dependencias.
