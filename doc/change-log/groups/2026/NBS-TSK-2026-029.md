# Resumen

Grupo que recoge la corrección del drift entre la primitive local `StarBorder` y la referencia oficial de React Bits.

## Objetivo del grupo

Volver a una base más fiel al componente fuente y dejar la personalización de marca en el consumo concreto del navbar.

## Cambios incluidos

- reimplementación de `StarBorder` según la referencia de React Bits
- adaptación del CTA `Contactar` con overrides transparentes de marca
- limpieza del drift documental sobre variantes locales que ya no existen

## Impacto en arquitectura / producto

La primitive reusable queda más cerca de upstream y el navbar mantiene el mismo lenguaje visual sin depender de una fork más agresiva del componente.

## Validación agregada

- `npm run lint`
- `npm run build`
- `npm run typecheck`

## Riesgos o arrastres

- si se reutiliza `StarBorder` en nuevos contextos, convendrá revisar si los overrides deben extraerse a una wrapper específica

## Notas para presupuesto

Refactor de bajo coste orientado a reducir drift y mantenimiento futuro.
